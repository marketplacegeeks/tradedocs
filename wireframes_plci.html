S07 — Create: Page 4 — Containers & Items
Purpose: Maker adds containers and items inline — no modals. Each container card has: dark field-header bars above input rows, an items table, and inline forms for add/edit.
Key elements: Info banner; Weight Unit selector; Container cards (numbered badge, field-header rows, items table, "+ Add Item"); inline edit row within items table; inline Add Item form below table; full-width "+ Add New Container" button; Next / Back buttons.
Three states shown: (A) Container 1 — saved, items present, one item in inline edit mode. (B) Container 2 — saved, no items yet, Add Item form expanded. (C) New container card appended above full-width Add button.

S10 — Create: Page 5 — Final Rates
Purpose: Auto-generated table showing one row per unique Item Code + UOM combination across all containers. Maker enters a single rate (USD) per row. All other columns are read-only. This is the last data-entry page; saving redirects to the Summary / Review page.
Key elements: 4-step wizard bar, read-only aggregated table (Item Code, Description, HSN, Total Qty, UOM), editable Rate (USD) column, calculated Amount (USD) column, validation alert if any rate is empty, Save & Review / Back buttons.

Purpose: Maker confirms they are ready to submit both the PL and CI for approval. Reminds user this covers both documents jointly.
Key elements: Confirmation text, PL No + CI No summary, Cancel / Confirm Submit buttons.
Submit for Approval

You are about to submit the following documents for approval:

Packing List: PL-2026-0004
Commercial Invoice: CI-2026-0004
Linked PI: PI-2026-0007

Both documents will be submitted together and can only be approved or rejected as a unit.

Cancel Confirm: Submit for Approval
S15 — Modal: Approve Confirmation
Purpose: Checker confirms approval of both PL and CI in a single action. Approval is final and moves both documents to Approved state.
Key elements: Document numbers summary, confirmation message, Cancel / Confirm Approve buttons.
Approve Documents

You are approving the following documents:

Packing List: PL-2026-0004
Commercial Invoice: CI-2026-0004

Both documents will move to Approved status. This action cannot be undone. The clean PDF will become available for download.

Cancel Confirm: Approve
S16 — Modal: Reject → Rework (with mandatory comment)
Purpose: Checker rejects both documents and sends them back to the Maker for rework. Comment is mandatory — the form cannot be submitted without it.
Key elements: Document numbers, mandatory rejection comments textarea (inline validation if empty), Cancel / Confirm Reject buttons.
Reject — Send for Rework

Rejecting the following documents and returning them to the Maker for rework:

PL-2026-0004 & CI-2026-0004
These comments will be visible to the Maker on the document.
Cancel Confirm: Reject → Rework
S17 — Modal: Permanently Reject (with mandatory comment)
Purpose: Checker/Admin permanently rejects both documents from any state. Terminal action — no further edits, submissions, or downloads possible on either document.
Key elements: Strong warning text (terminal/irreversible), document numbers, mandatory comments textarea, Cancel / Confirm Permanently Reject buttons (destructive style).
Permanently Reject
⚠ This is a terminal action. Both documents will be permanently rejected and cannot be edited, resubmitted, or downloaded. This cannot be undone.

Permanently rejecting:

PL-2026-0004 & CI-2026-0004

S19 — Modal: Copy Container Confirmation
Purpose: Maker confirms copying a container. All items are copied; Container Reference, Marks & Numbers, and Seal Number are left blank; Tare Weight is pre-filled. Maker must then fill in the blank fields on the copy.
Key elements: Source container summary, copy behaviour description (what is copied vs. left blank), Cancel / Confirm Copy buttons.
Copy Container

Copying container:

Container Reference: CONT001
Items: 2 items (ITEM-001, ITEM-002)
Tare Weight: 2.200 kg
What will be copied:
  • All items (Item Code, Description, HSN, Qty, Weights, etc.)
  • Container Tare Weight (pre-filled; you can adjust)
What will be left blank (you must fill in):
  • Container Reference
  • Marks and Numbers
  • Seal Number
Rates are not set per container — they apply at the Final Rates step.
