# Phase 3 Plan 02: Wire get_permissions() on All Document Viewsets

<objective>
Replace the blanket `permission_classes = [IsAnyRole]` on every document viewset with a `get_permissions()` override that enforces role-based access at the DRF routing layer. Also apply `throttle_scope` to creation endpoints on PI, PL, and COA viewsets.

Purpose: Currently Checkers can reach `perform_create/perform_update/perform_destroy` and are only blocked by a manual role check inside those methods. `get_permissions()` moves the enforcement to DRF's routing layer so Checkers receive HTTP 403 before the view method body executes — a defense-in-depth improvement.

Output: 4 modified view files. Existing manual role checks inside `perform_*` methods are retained as a second layer of defense; `get_permissions()` adds the first layer.
</objective>

From apps/accounts/permissions.py (after plan 01):
```python
class IsAnyRole(BasePermission):      # all authenticated users — use for reads
class IsMakerOrAdmin(BasePermission): # MAKER + COMPANY_ADMIN + SUPER_ADMIN — use for writes
class IsSuperAdmin(BasePermission):   # SUPER_ADMIN only — hard-delete endpoints keep this
```

Permissions matrix (from NOTES.md):
- CREATE, EDIT, DELETE: MAKER + COMPANY_ADMIN + SUPER_ADMIN only (NOT Checker)
- LIST, RETRIEVE, PDF, AUDIT LOG: all authenticated roles (IsAnyRole)
- WORKFLOW actions: all roles can POST (WorkflowService validates role internally)
- SIGNED COPY upload: all roles can upload (IsAnyRole)

DRF get_permissions() pattern (from CONTEXT.md specifics — IsAuthenticated must be explicit
because get_permissions() overrides the global DEFAULT_PERMISSION_CLASSES):
```python
from rest_framework.permissions import IsAuthenticated

def get_permissions(self):
    if self.action in ('create', 'update', 'partial_update', 'destroy'):
        return [IsAuthenticated(), IsMakerOrAdmin()]
    return [IsAuthenticated(), IsAnyRole()]
```

Why IsAuthenticated() must be included: When get_permissions() overrides the global
DEFAULT_PERMISSION_CLASSES, the global IsAuthenticated default is no longer applied.
Omitting it would allow anonymous requests to hit list/retrieve endpoints. CLAUDE.md rule #10
requires every DRF view to explicitly declare permission_classes — this is the per-action
equivalent.

ScopedRateThrottle usage (from DRF docs):
- Add `throttle_scope = 'document_creation'` as a class attribute on viewsets
- DRF automatically applies rate limiting only when the scope matches DEFAULT_THROTTLE_RATES key
- The scope applies to ALL actions on the viewset — this is intentional (100/day is generous)

CI viewset note: CI does NOT have create (disabled with ValidationError) or destroy (disabled).
PATCH (partial_update) on CI is the only write action. This must be restricted to IsMakerOrAdmin.
CI line item viewset: only partial_update is allowed (create/destroy raise ValidationError).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Unauthenticated → any endpoint | get_permissions() now explicitly returns IsAuthenticated() — anonymous requests get 401 before role check |
| Checker role → write endpoints | Checker is authenticated but must not modify documents |
| Maker role → other users' documents | Maker can only edit documents they created (enforced by creator check in perform_update) |
